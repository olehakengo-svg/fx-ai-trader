### 診断
## 原因分析

### 推測（確認前）

**STATUS API タイムアウトについて**
- APIサーバープロセスがハング・フリーズしている可能性
- サーバーリソース枯渇（メモリリーク、CPU過負荷、ディスク満杯）によるレスポンス不能
- ネットワーク層の障害（ロードバランサー、プロキシ、ファイアウォールのセッション詰まり）
- Pythonプロセスがデッドロック状態に陥っている可能性

**トレード0件・全モードWAITについて**
- APIが死んでいるためステータス取得自体が不正確（偽陽性の可能性大）
- ただし仮にプロセス自体は生きていても、フィルター（スプレッド・ボラティリティ・時間帯）が全条件を遮断している可能性
- ブローカーへの接続（MT4/MT5またはREST）が切断されたままになっている可能性

---

### 確認事項（優先順）

**サーバー死活確認（最優先）**
- [ ] サーバーへSSH直接ログインできるか
- [ ] `top` / `htop` でCPU・メモリ・ロードアベレージ確認
- [ ] `df -h` でディスク残量確認
- [ ] 対象プロセスが存在するか（`ps aux | grep [プロセス名]`）

**プロセス・ログ確認**
- [ ] アプリケーションログファイルを直接確認（ログ`[]`はAPI経由取得失敗のため実ファイルを見る）
- [ ] システムログ（`/var/log/syslog` 等）にOOM killerやパニックの痕跡がないか
- [ ] プロセスがゾンビ化していないか

**ネットワーク・接続確認**
- [ ] ブローカーAPIへの疎通確認（`ping` / `curl`）
- [ ] APIサーバーのポートがLISTENしているか（`ss -tlnp`）
- [ ] タイムアウト値の設定が短すぎないか（暫定的な問題の場合）

---

### 対処方針

1. **まずサーバーに直接SSHして生死確認** — 監視系がAPIに依存しているため、監視ツール側の結果を盲信しないこと
2. リソース枯渇が確認された場合 → 不要プロセス停止、ディスク掃除後にサービス再起動
3. プロセスがハング確認された場合 → `kill -9` 後に再起動、再起動後の復帰ログを必ず

### 診断
## 原因分析

### 【推測】含み益がShadowに留まりLIVEに昇格しない根本原因

**1. TREND_BULL TF Bypassによる強制Shadow化**
- `post_news_vol×EUR_USD`が `TREND_BULL WR=0% → shadow` でShadow送りされている
- `daytrade_gbpusd:regime_trend_bull_dt_tf` のブロックカウントが67と最多クラス
- EUR_USD / GBP_USD共に「上位TFのTREND_BULL判定」がLIVE昇格を阻んでいると推測

**2. SCORE_GATEによる連鎖ブロック**
- `daytrade:score_gate` 136件、`daytrade_gbpjpy:score_gate` 79件
- USD_JPYの`streak_reversal`がscore=-3.00で連続ブロック → daytrade系全般でLIVEシグナルが出ていない状態

**3. `PAIR_PROMOTED`未設定**
- Shadow含み益が+49.8pipに達しているにもかかわらず、当該ペアがPAIR_PROMOTEDリストに入っていないため、自動昇格ロジックが発火しない

---

### 【確認事項】対処前に確認すべきこと

- [ ] `TREND_BULL WR=0%` の根拠データが現在も正しいか（上位TF足の取得・計算が正常か）
- [ ] `regime_trend_bull_dt_tf` の判定閾値設定値と、現在のEUR_USD / GBP_USDの実際のレジーム状態が乖離していないか
- [ ] PAIR_PROMOTEDの昇格条件（pip閾値・継続時間）の現在設定値
- [ ] `main_loop_restarts=1` の再起動タイミングとShadowポジション生成タイミングの前後関係（再起動後にLIVEステートが初期化されていないか）
- [ ] `score=-3.00`（USD_JPY daytrade）のスコア算出元データ異常の有無（`unknown`ラベルのブロックも散見されるため、スコアソースのNaN/未初期化を疑う）

---

## 対処方針

### 即時対応

- **`post_news_vol×EUR_USD` および `sr_break_retest×GBP_USD` をPAIR_PROMOTEDに追加**し、Shadow→LIVE昇格を手動でアンロック
- `TREND_BULL WR=0%`の上位TFデータを手動取得し、計算結果が実態と一致しているか突合確認

### 短期対応

- `regime_trend_bull_dt_tf` ブロックロ
